# 작업지시서 — 랜딩 페이지 (Claude Code 용)

> 저장소 위치 권장: `docs/landing-page-tasks.md`
> 디자인 기준: `docs/landing-mockup.html` (설계 시안). 이 목업의 레이아웃·카피·색/타이포 토큰을 그대로 구현한다.
> 기존 규약 계승: React+Vite SPA, 같은 오리진 서빙(`static/`), `SpaForwardingController` 딥링크 forward, 프론트 수정 후 `npm run build` 산출물 커밋.

## 배경 — 왜 만드나

지금 앱은 접속 시 바로 로그인 화면이라 **"파는 화면"이 없다.** 랜딩 페이지가 첫 세일즈 자산이다. 포지셔닝은 경쟁사(셀러팟)의 넓은 "올인원 수익 관리"와 달리 **뾰족한 한 문장**으로 간다:

> **"ROAS는 높은데 팔수록 손해인 상품을, 정산 실수령 기준 진짜 순이익으로 잡아낸다."**

## ⚠️ 절대 규칙

- **가짜 사회적 증거 금지.** "누적 거래액 ₩00억", "셀러 0000명", "만족도 4.9" 같은 수치를 **지어내지 말 것.** 현재 실사용자 0명이다. 대신 목업처럼 **제품 인사이트(적자상품 명세서)** 로 신뢰를 만든다. "얼리액세스/베타" 프레이밍은 허용.
- 가격은 실제 플랜(`/api/plans`)과 일치시킨다. 임의 가격 표기 금지.

## T8 — 랜딩 라우트 구현

### 8.1 라우팅 재구성
- **`/` = 공개 랜딩 페이지**(비로그인 접근 가능). 현재 `/`의 대시보드는 **`/dashboard`로 이동.**
- 인증 벽(`auth.jsx`) 조정: 공개 경로(`/`)는 벽에서 제외. 보호 경로는 `/dashboard`, `/ad-roi`, `/accounts` 등.
- **로그인 상태에서 `/` 방문 시 `/dashboard`로 리다이렉트**(로그인 유저는 랜딩 대신 앱으로).
- `App.jsx` 라우트 + `SpaForwardingController`에 `/`(및 필요한 앵커) forward 추가. 내부 링크(대시보드→`/dashboard` 등) 전부 갱신.

### 8.2 랜딩 컴포넌트
- `frontend/src/pages/Landing.jsx`(+ 전용 스타일). 목업의 섹션 순서 그대로: **nav → 히어로(정산 명세서 시그니처) → 문제 스트립 → 가치 4카드 → 요금제 → 최종 CTA → 푸터.**
- 목업의 디자인 토큰(색/폰트/그림자)을 **랜딩 스코프로 격리**한다. 앱은 다크 테마이므로 랜딩의 페이퍼(밝은) 스타일이 앱 전역 CSS로 새지 않게 래퍼 클래스 하위로 한정하거나 별도 CSS 모듈 사용.
- 폰트: Pretendard(본문) + Space Mono(숫자·명세서). 목업의 CDN 링크 사용.

### 8.3 동적 연결
- **요금제 섹션은 `/api/plans`에서 가격을 받아** 렌더(하드코딩 금지, 플랜 정책 한 곳 유지).
- CTA 버튼: "무료로 시작"·"무료로 내 순이익 보기" → `/signup`, "로그인" → `/login`.
- nav는 인증 상태 반영: **비로그인** = 로그인/무료 시작, **로그인** = "대시보드로" 링크.

### 8.4 품질 기준(기존 규약)
- 반응형: 모바일 폭(≤820px)에서 히어로·카드·요금제가 1열로 스택(목업 미디어쿼리 참고).
- 접근성: 키보드 포커스 표시(`:focus-visible`), 명세서 카드에 `aria-label`, `prefers-reduced-motion` 존중(리빌 애니메이션 비활성).
- 히어로 카피·숫자는 목업 그대로(적자상품 B: 매출 270,000 − 원가 405,000 − 기타 4,737 − 광고 20,000 = −159,737, ROAS 13.5×). 이는 실제 시드 데모와 정합.
