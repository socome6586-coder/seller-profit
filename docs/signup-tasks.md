# 작업지시서 — 회원가입 화면 보강 (Claude Code 용)

> 저장소 위치 권장: `docs/signup-tasks.md`
> 실제 코드 확인 후 작성(추정 아님): `frontend/src/pages/{Landing,Signup,Login}.jsx`, `Landing.css`, `styles.css`.
> 기존 규약 계승: 프론트 수정 후 `npm run build` 산출물 커밋 + **백엔드 재시작 필요**(CLAUDE.md 129번째 줄 참고).

## 배경

지금 회원가입(`/signup`)은 통일된 페이퍼 브랜드(`styles.css` `:root` 토큰) 위에 `.auth-card` 하나만 중앙에 떠 있는 구조라 비어 보인다. 폼 필드를 늘리는 방향은 **금지**(전환율 저해) — 대신 랜딩의 정산 명세서 시그니처를 재사용해 "왜 가입하는지"를 다시 보여주는 2단 레이아웃으로 바꾼다.

## ⚠️ 절대 규칙

- **회원가입 폼 필드는 늘리지 않는다.** 지금처럼 이메일+비밀번호만. 이름/전화/회사명 등 추가 금지.
- **가짜 지표 금지**(기존 landing-page-tasks.md 규칙 계승). "셀러 N명" 등 지어내지 않는다.
- 새로 넣는 카피의 사실 확인: "카드 등록 불필요"(가입 시점엔 결제 수단 요구 안 함, `billing/` 은 별도 구독 단계), "AES-256 암호화"(`crypto/AesGcmEncryptor` 실존, 단 이건 **연동 API 키** 암호화이지 회원가입 자체와는 무관하니 문구는 "쿠팡 연동 시 API 키는 AES-256으로 암호화됩니다" 식으로 정확히 쓸 것. "회원가입 정보가 암호화된다"처럼 오해되게 쓰지 말 것).

## T11.1 — ReceiptCard 공용 컴포넌트로 추출

- `Landing.jsx` 안에 인라인으로 있는 정산 명세서 마크업(`.receipt`/`.receipt-top`/`.r-name`/`.roas`/`.r-body`/`.r-line`/`.r-total`/`.r-flag`)을 `frontend/src/components/ReceiptCard.jsx` 로 추출.
  - props 화: `productName`(기본 "적자상품 B · 이번 달"), `roas`(기본 "13.5×"), `lines`(배열: `{label, value, sign}`), `totalLabel`, `totalValue`, `flagText`. 기본값은 기존 랜딩 카피 그대로(매출 270,000 / 원가 −405,000 / 기타 −4,737 / 광고 −20,000 / 진짜 순이익 −159,737 / "적자 · 광고 켤수록 손해!").
  - CSS: 현재 `Landing.css` 의 `.landing .receipt...` 선택자들을 **`.landing` 접두사 없이** 공용으로 뺀다(새 파일 `frontend/src/components/ReceiptCard.css` 또는 `styles.css` 에 추가). Landing.css 쪽 규칙은 삭제하거나 새 공용 클래스를 import 하도록 정리해 **중복 정의가 남지 않게** 한다.
- `Landing.jsx` 는 이 새 컴포넌트를 사용하도록 교체.
- **AC:** 랜딩 히어로가 기존과 시각적으로 동일하게 렌더(회귀 없음). `ReceiptCard` 가 `Signup.jsx` 에서 import 가능.

## T11.2 — Signup 2단 레이아웃

- `Signup.jsx` 를 2단 그리드로 재구성(데스크톱: 좌 폼 / 우 가치 패널, 모바일: 세로 스택 — 폼이 먼저 보이도록 순서 유지). **폼 자체(이메일/비밀번호 input, 검증 로직)는 그대로 재사용**, 레이아웃만 감싼다.
- **우측 패널 구성** (신규):
  1. `ReceiptCard` (props 그대로, 랜딩과 동일한 적자상품 예시 — 새 이야기 지어내지 않음).
  2. 아래 3가지 안내를 짧게:
     - **다음 단계**: "가입 → 쿠팡 계정 연동 → 진짜 순이익 확인" (3단계, 아이콘 불필요, 텍스트로 충분)
     - **필요한 것**: "쿠팡 판매자 계정만 있으면 시작할 수 있어요."
     - **안심 문구**: "무료로 시작 · 카드 등록 불필요 · 연동 시 API 키 AES-256 암호화"
- 좌측 폼 카드는 기존 `.auth-card` 스타일 유지(변경 최소화), 전체를 감싸는 새 래퍼만 추가(`.auth-split` 등 네이밍은 자유, 기존 `--bg`/`--panel`/`--line` 토큰 재사용, 새 색상 도입 금지).
- **AC:** `/signup` 접근 시 좌(폼)·우(가치 패널+ReceiptCard) 2단 렌더. 기존 가입 동작(검증·에러·로그인 이동) 무변화. 모바일 폭에서 1열 스택, 폼이 먼저.

## T11.3 — (선택, 여력 되면) Login 페이지도 가볍게 정렬

- `Login.jsx` 는 폼 자체는 그대로 두되, 좌우 여백 밸런스만 `Signup.jsx` 와 톤 맞춤(굳이 ReceiptCard 반복 배치는 하지 않음 — 이미 가입 유도가 목적이 아니므로 과함). 시간 없으면 스킵 가능.
